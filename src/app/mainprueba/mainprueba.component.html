Angular Routing para Dummies

El routing de angular nos permitirá habilitar un sistema de navegación en nuestra App. Por ejemplo, si nuestra App tiene un módulo de login donde debemos introducir user y password tendremos probablemente un myapp/login y un myapp/dashboard para cuando hayamos logeado satisfactoriamente. En resumen, el Router permite que podamos mostrar ciertos componentes dependiendo de la url en la que estamos. También podremos enviar informacion a través de la url y podremos evitar que un usuario,no autenticado acceda directamente al myapp/dashboard (en este ejemplo). Para poder utilizar el routing, en el momento que creamos el proyecto angular le debemos decir que si queremos usar el routing para que nos genere el archivo app-routing.module.ts. 


            import < NgModule > from '@angular/core';
            import < Routes, RouterModule > from '@angular/router';
            
            const routes: Routes = [];
            
            @NgModule(<
              imports: [RouterModule.forRoot(routes)],
              exports: [RouterModule]
              >)
            export class AppRoutingModule < >

Crearíamos los componentes adicionales para movernos entre ellos:
Creamos el dashboard con: $ng g c dashboard
Y despues creamos la view1 con: $ng g c view1
El siguiente paso será modificar el archivo app-routing.module.ts

const routes: Routes = [
            < path: 'dashboard', component: DashboardComponent >,
            < path: 'view/:id', component: View1Component >, ];

De esta forma cuando la url tenga /dashboard/ cargue el componente DashboardComponent y cuando la url tenga /view/2 va a cargar el componente View1Component y va a enviar un parametro id.
